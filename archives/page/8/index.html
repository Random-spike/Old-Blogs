florr商店最近很不稳定。

商店定价有以下几个因素

1.稀有度（就是说卡获取难度，典型：以前的铀和刀）

2.实用度（就是说卡的价值，典型：bubble）

3.导向（就是说m2b希望你买这些让他赚钱：典型：bur，sand，light）

接下来就是对现状分析。

上图为这次商店

下图为昨天商店

看到了web降价，速涨价。以及回血和sponge涨价。

初步推测：

在即将到来的大更新之后。蜘蛛会变得更加稀少。不会出现类似wave时期的mob狂潮。

同时持续回血（leaf）和海绵价格上升可能暗示了有可能受到的更多伤害（需要持续回血配合海绵使用）

值得注意的是：fangs从1变成2，rose2变成3.在这个抢s不需要回血的时代，显然是格格不入的。因此暗示了可能的大更新内容。

rock大幅度涨价使不出意料的。bubble不涨价也是不出意料的（建议不要下手太多ububble，但mbubble打了半价在你熟悉垫卡之后是退建冲的）

cutter掉价是否意味着蚂蚁窝的更加频繁出现？（铀涨价1倍说明可能回到wave时期铀掉率）

挂机时请在“生火间”挂机（否则没有随机事件）。

注意：无论何时的游戏都要遵守一个准则：所有物资保持正增长。 这到后期可能比较难，你必须精密安排职业。

本攻略不是速通攻略。  link

警告：阅读此攻略可能会降低你的游戏乐趣。 建议通关1次后再阅读本攻略。  link

如果游戏进度停滞不前（比如我第一次游戏的时候卡在皮套不动了，直到遇到侦察兵买了地图才知道有铁套钢套），也可以阅读此攻略。  link

完全原创，难免会有疏漏，有问题请在评论区指出。

前期：

首先生火，然后添柴。

会有一个陌生人进来，不用管他。

能伐木了之后能伐就伐，等陌生人醒来就不用添柴了。

陷阱先只造一个，如果大型野兽摧毁陷阱就有概率得到100肉和100毛皮，还是比较值的。

能造货车了之后先造货车，因为它可以让你的伐木数量从10提升到50。

能造小屋了之后先全力发展人口，同时保证陷阱有且只有一个，至于其他的先不管。

可能会出现“the shady builder”，如果你同意，他会带走你300木材，然后为你建造一间小屋（builds a hut）或带着木材跑路（has made off with your wood）。

人口 ≥ ≥ 40之后可以考虑多造几个陷阱。

人口满了之后（80人）全力造陷阱。

之后直到通关，保证你的小屋和陷阱顶着上限。人口不用担心，会自己恢复。定期查看陷阱。

之后造各种职业建筑，同时安排2个猎人和1个陷阱师。这可以让你的陷阱变成上饵陷阱，可以提高效率。

造完了之后造罗盘，解锁“漫漫尘途”，进入中期。

中期：

先制造皮革，造双肩包和皮甲，其他的先不造。同时多点猎人和熏肉师，把熏肉的量提起来。

能力 先只带一个熏肉出发，随便在村庄

A 周围走两步，很快你就会饿死。饿死十次可以得到“减缓代谢”能力，让你从两步一肉变成四步一肉。

之后按同样的方法渴死十次，可以得到“荒漠跳鼠”能力，让你从一步一水变成两步一水。（注意带至少2个熏肉，以免饿死。当你渴死的时候熏肉正好吃完）

同时如果遇到怪物，就先徒手撸它。撸50次可以获得“拳击手”能力（一拳2血），150次“武斗家”能力（一拳6血）， 300次“徒手宗师”能力（一拳12血，速度翻倍）。这很不好刷，但是如果刷出来，你基本就无敌了（12血比任何一个武器的伤害都高，速度也更快，因此后期建议徒手出门。）可以在之后刷，在探索时打着怪物就刷出来了（“徒手宗师”甚至可以在后期刷），但是建议至少刷出“拳击手”。

以下能力可以在探索的同时刷，甚至可以不全部刷。

如果出现小偷，不用担心，重新安排一下职业，保证物资正增长，在“小镇”挂机，一段时间后小偷会被抓住。之后放掉小偷，这样可以得到“潜行”能力（怪物减半）。后期怪物超级多，这个技能还是很有用的。如果杀掉小偷会得到一些木头之类的，但没有这个能力来的划算。小偷不像宗师和侦察兵，小偷只会出现一次。 有可能会有宗师出现，他一次能教你一个能力，总共能教你三个能力：“避实就虚”（怪物失手的概率增加）， “精密”（自己失手的概率减少），“野蛮人”（近战武器造成更高伤害。然而还是超不过“徒手宗师”的伤害）这三个技能都比较有用，因此请常备火把（之后探索也需要大量火把，所以请造至少20个火把）。

也可能有侦察兵出现，能学“侦察术”能力就先学，之后购买地图。大量储存毛皮和鳞片，能买就买。地图买完了（全地图全部解锁）后游戏会卡住，刷新一下就行了。（不一定非要买了地图才开始探索，你甚至可以一个地图不买，但是建议通过买地图的方式解锁地图，这样探索时更有针对性。）

此时一共十个能力。（可以不用刷完所有能力，比如那三个提高攻击力的能力可以在之后刷，但是建议至少刷出“拳击手”。）

探索

造出水壶，皮甲（骨枪可造可不造，建议不造），开始探索。解锁周边的地图，刷自己的徒手能力，以及在挂机时刷其他能力。

地图图例：

H 小屋，里面可能有怪物，会有一些物资

V 洞穴，里面有怪物、物资。需要火把

O 小镇，里面可能有怪物，会有一些物资。需要火把

Y 城市，类似小镇，可能不需要火把

P 前哨，攻下V，O或Y会出现。你可以在那里填满你的水壶，同时获得3-9个熏肉。每次探险只能使用一次

I 铁矿，攻下后解锁铁矿工人。需要火把

C 煤矿，攻下后解锁煤矿工人，炼钢坊（解锁炼钢工人）。需要火把

M 沼泽，会有一老人，带符咒去可以获得“美食家”能力（一肉恢复8血变一肉恢复16血）

S 硫磺矿，攻下后解锁硫磺矿工，军械坊（解锁军械工人，造子弹）。需要火把

W 坠毁星舰，探索后解锁“破旧星舰”

F 战场，没有怪物，会有一些武器和外星合金

B 巨坑，类似于战场

小径，攻下V，O，Y，I，C，S，W之后会出现，连接村庄。上面没有怪物

先在周围逛逛，攻下周围的V和O（村庄周围一般没有Y）。之后攻I，造出铁套。攻下C，造出钢套。

不建议在这个时候攻S，里面的怪物实在太强大，最好等进入后期。

如果觉得时机成熟了可以带上1个符咒去M，获得“美食家”能力。（知道陷阱的重要性了吧）

钢套造出来之后，进入后期。

后期：

战斗建议： 这个时候徒手出门就行了，不必带武器。

有一个东西很实用：套索。

它可以让你的敌人眩晕一阵子，你可以抓住这宝贵的时间猛烈攻击。（比如攻S最后一个65血的怪物时，它可能会派上大用场）

而且它非常轻（0.5），价格也十分便宜（10牙齿）。

至于其他武器，自己不要造，出门不要带，但如果探险时捡到就放在村庄里。

硫磺矿可攻可不攻。同时没有必要把所有的V O Y都变成P。

到W处去一趟，解锁“破旧星舰”。你已经离通关很近了。

多到F或B去，可以获得很多武器和外星合金。（这个时候能不死就不死。）

可以考虑去的时候攻下一个V O Y把它变成P，回来的时候经过它。（此法在攻或去任何一个较远的地方时都适用）

不建议自己买外星合金，那太贵了，你可能需要为一个合金挂上1小时机。

探索完整张地图上的F和B（不探索完也行），你差不多能得到20个外星合金。两三个升级引擎，剩下的全部加固船身，直接点火起飞。（不要在意那个提示，如果你死了会回来的。如果你舍不得你繁荣昌盛的村庄，请在起飞前导出并备份存档）

我的存档

起飞后直接靠边，边上陨石最少，甚至一个没有。（我喜欢靠左边）之后基本不用管，甚至可以挂机。

恭喜通关。

1- 引子

以前经常和大家介绍为什么小学生，初中生，高中生要学信息学奥赛。

答案只有一个：升学。

必然是为了升学啊，难不成是为了成为技术宅然后拯救世界么。

那今天我来问个问题：

您觉得，这些信息奥赛生，在高中毕业之时，高考之后，大学选什么专业？

要说是考古，我肯定是不信的。

再猜。

计算机？

没错，就是计算机科学系。

按照国内现在的学科分配，大一应该是信息大类，之后，按照成绩分，成绩优秀的去计算机科学系。

简称计科。

计算机科学系的一个主要学习内容，就是算法。

信息奥赛生在高中时期最擅长的，也是算法。

所以对于这些曾经的奥赛生来说，这部分理解起来很简单。

唯一的区别，就是，大学的要求不只是写出程序的实现，更要证明算法复杂度。

这也不是什么难事儿，随便学学就满分了。

然而，算法，只是计算机科学系中教授内容的一部分。

否则，岂不是信息奥赛生可以跳过大学直接工作了么。

这就要提及在信息学奥赛中一个被忽视的部分，也是在计科中非常重要的，堪比算法的内容，那就是：

软件工程。

如果光会算法，完全不会软件工程，或者听了软件工程就恶心，那我保证你连个大厂实习都找不到。

这是个事实，不需要争辩：

1- 如果你是个正经的奥赛生，应该拿个提一没问题

3- 如果拿了提一，你应该能上个一本的计科也没问题

3- 如果你上了个正经的计科，没道理你没学过软件工程

4- 如果你在大学学了软件工程，却完全没学会，说明你挂科了

5- 如果你挂科了，你肯定过不了大厂的人力资源部门的秘书小姐姐的筛选

6- 1到5好好走几遍，看看这个逻辑是不是缜密。

所以软件工程就是这么的重要。

说不可或缺，也没有什么不妥的。

软件工程里面有个重要的方法论，那就是面向对象编程，英文是Object Oriented Programming，简称OOP。

从开始带信息学奥赛这一天算起，我就一直觉得，信息学奥赛，是可以用计算机科学中的软件工程来化解难度的。

只不过，提高组通常与软件工程无关，纯拼算法。

只是，今年的提高组，偏偏就来了这么一道题大模拟，主题就是OOP。

[CSP-S 2023] 结构体 题解

在上纽（上海纽约大学）学了 python 的面向对象编程（OOP），我发现了信息竞赛题目（尤其是这道题）和 OOP 之间的紧密联系。

在学校的 OOP 相关题目往往都是老师提供程序的类和方法框架，然后我们向其中编写代码来解决问题。例如上学期的 Data Structure，我们的栈、队列、哈希表、二叉树等数据结构都是完全按照 OOP 的方式编写的。

信息竞赛与之不同的是，我们没有已有的框架，于是我们需要仔细分析题目，来决定自己需要定义什么类型、变量以及方法，然后再编写具体的代码来解决问题。这样做题思路会比上来就开始写代码要清晰很多。以及信息竞赛使用的语言是不足够 OOP 的 C++ 语言，而且处理的任务很小，目标也是代码效率以及编程效率至上，故很多时候使用较为简单的 struct 以及全局函数即可。

本题可以当作一个 OOP 练习题。所有的类型以及元素都用 struct 存储。

用一个 struct 存储类型，根据题意，需要存储这个类型的大小，对齐规则，和其成员列表。为了方便快速寻找类型内特定名称的成员，可以额外存储一个名称到成员列表位置的 map。
