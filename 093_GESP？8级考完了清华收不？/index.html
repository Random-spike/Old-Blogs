GESP？8级考完了清华收不？ 新文
2023-10-27 - /093_GESP？8级考完了清华收不？/

明天考GESP，就是那个，号称可以免掉CSP-J初赛的计算机学会的考级。

您，交了300块钱+的高价考试费，让孩子去考试了吗？

如果您交了钱，并且一会儿就打算让孩子去，那，我表示深切的同情。

因为大几率，结果会让人相当不满。

所以，如果您觉得我是来宣传GESP的，那就大错特错了。

在这次9月的GESP考级中，若是还有任何机构在宣传并且要求参加的话，从家长角度来看，那就非常可以质疑：你们到底懂不懂？

先说费用：

这是个简单的等差数列，用高斯的方法就好了，（首项+末项） * 项数 / 2，一共是2960。

关键是，一次还不一定过。

所以，搞不好是2960 乘个3，这还是保守估计。

再说难度：

我上次写GESP，是6月以前，是在GESP官方明确表示，只要通过8级，就一定能免初赛之后，的这个测试期。

经过测试，这个考级，完全不像声称的一样有用。

从一开始我就说，GESP唯一存在的价值，就是能够免考初赛。

如果这个功能达到不了，那就没有任何存在的意义，

一般来说，考级总是不敢出太难的题的，考级考级，如果两三次都考不过，那考的是什么级？

所以，这从数学模型角度上，似乎是成立的：若干次简单考试（GESP），可以抵消掉一个难的考试（CSP-J的初赛）。

但现在，显然GESP给自己的定位，是青出于蓝，而胜于蓝的状态。

用另一个方式来说，那就是：

它是个，比CSP-S组还难，的，一个，路子野的比赛。

也就是说，它非常不满意自己是个简单考试的定位。

它非得上天。

路子野，说得是，CSP-S的初赛都不敢这么考，嘿，它就敢。

直接切入主题好了：

有在2023年CSP-J组几乎满分的同学，参加6月份的4级没过。

这里不排除GESP考试当天没重视，题目分布不熟悉，以及最终两道重磅大题没发挥好的可能。

这里也不排除学生个体的，身体状况差异的问题。

但是，我再说一遍：

CSP-J组几乎满分的同学，4级过不了。

别说北京了，这种J组考试初赛90+的孩子，就是在环境极其恶劣的浙江，都过了。

一个初赛完全没有问题的奥赛生，想要双保险，所以考了4级，但很遗憾的。

就是没过。

说清楚这里的逻辑悖论了吧？

1- 4级一定比8级简单

2- 4级，比CSP-J要难，因为题量大，同时时间还短。

3- 4级过不了，8级能过？

4- 考GESP是为了不考J组初赛，如果能考过J组，就不用参加；如果参加，是因为害怕过不了J组

5- J组初赛50元一年，考20年也不过才1000元

6- 考一套CESP 2960，算个车马费停车费，3000元，那就60年的J组考试，足够从11岁，一直考到七十古稀

7- J组初赛能拿90+，初赛100%过；4级不能100%过。

8- 从1开始继续重复100遍，不断提出质疑 “为什么，我，要，考这个？”

所以综上，为什么我闲的没事儿了要去考个GESP？

看到6月的题之后，如果还有机构愿意送孩子去当炮灰，那么，我也是无话可说。

只是，这是有钱烧得，不花难受？

我相信没有人家里钱是大风刮来的，能随便这么2960一套，来个3套，把考180年的CSP-J的钱就这么都扔水里。

就算愿意扔水里，也要能听个响对吧？

要是CSP-J能考90+的孩子，都有可能过不了四级，那您还要考CESP吗？

任何人都能看出来这里面逻辑关系是混乱的吧？

显然是。

所以，你这个GESP到底是干啥用的？

用途不明。

所以这个GESP的路子，还是很野的。

只是，我说路子野，说的是题路野。

没说是野路子。

这个东西还真就不是个野路子，这是个嫡系的产品线，来，看看这个东西定义：

CCF编程能力等级认证，英文名Grade Examination of Software Programming（以下简称GESP），由中国计算机学会发起并主办，是为青少年计算机和编程学习者提供学业能力验证的平台。GESP旨在提升青少年计算机和编程教育水平，推广和普及青少年计算机和编程教育。GESP覆盖中小学全学段，符合条件的青少年均可参加认证。GESP考察语言为图形化编程、Python编程及C++编程，主要考察学生掌握相关编程知识和操作能力，熟悉编程各项基础知识和理论框架，通过设定不同等级的考试目标，让学生具备编程从简单的程序到复杂程序设计的编程能力，为后期专业化编程学习打下良好基础。

画一下重点：

1- CCF编程能力等级认证

关键词：CCF （同样还有CCF的从属关系的，那就是CSP-J）

2- 中国计算机学会发起并主办

关键词：发起+主办

3- 是为青少年计算机和编程学习者提供学业能力验证的平台

关键词：能力验证（竞赛）

4- GESP覆盖中小学全学段

关键词：小学

5- 主要考察学生掌握相关编程知识和操作能力，熟悉编程各项基础知识和理论框架

关键词：基础知识

6- 让学生具备编程从简单的程序到复杂程序设计的编程能力，为后期专业化编程学习打下良好基础。

关键词：后期，专业化

1和2，关键词：CCF （同样还有CCF的从属关系的，那就是CSP-J） + 发起+主办

说明这是嫡系官方。

3，关键词：能力验证（竞赛）

说明，这是竞赛。

然而，它声称自己是个考级，声称自己的考级8级过了，就可以抵消更难的J组初赛。

然而，再次然而，它的4级，就比J组初赛难。

推出矛盾。

矛盾之一。

4- 关键词：小学

加上123，说明这是面向小学的竞赛

好吧，考级。

然而，第三次然而，它的六级大纲，是这样的：

二叉树排序树，这说的应该是二叉搜索树，BST。

这东西在提高组，都是不一定考的东西。

深度优先搜索，DFS，这在提高组的复赛，是必考的。

但是，你不是说这是面向小学的吗？

现在我国的小学生都这么厉害了吗？

在9年义务教育时期，就已经能普及深度优先搜索啦？

那这可真是个重要的全国范围的科普成果。

只可惜，事实是，在北上广，连普通的高中生，都还不知道C++怎么写个“Hello World”。

所以，又来一个矛盾。

矛盾之二。

5，6- 关键词：基础知识 + 后期，专业化

说明这是个大学的预备课。

没错吧？基础知识，那就可以在大学之前的课；所谓后期，专业，这不就说的是大学专业课吗？

好的。

先都不说八级，我们看看七级是什么：

首先要指出的就是，这里面的内容混乱无比。

常用函数，这些简单极了。

哈希表，简单极了。

图论，开始难了。

欧拉大神，1736年才用一篇论证七桥问题的论文开创了图论。

后面花了100年的时间，人们才创造出了树型结构的概念。

直到最近60年，在Dijkstra的研究帮助下，图论才有了一个长足进步。

这三个东西，是怎么能放在一起的？

这还不是最离谱的。

最离谱的是动态规划。

动态规划，这是国内普通一本根本不开的内容。

动态规划，在麻省理工的主课，《算法导论》中，要讲整整4节课。

一节1个小时不灌水。

咱们先不说学术上超英赶美，您这现在小学生，就要脚踢麻省理工啦？

咱也不知道，咱也不敢问。

总之，您圣明。

至少我能说，这不是大学预备课。

这是矛盾之三。

最后，我们再看看8级。

光有个名字，没见到题，不清楚具体是什么。

但这倍增，难不成您说的是倍增求LCA？也就是最大公共祖先的倍增法？

您难道不知道这提高组里面也是不好理解的难题？

不过八级上点儿难题没问题。

只是，后面的算法的时间空间效率分析就有点扯了，都到8级了，你跟我说，算法优化？？

就好比，微积分都考完了，你问我加法会不会？
