0- 引语

CSP-J组的考试眼看就到，就是这个周六了。

例行的，每年考试前一天，都押个题。

至于为什么考前押题，那是因为太忙，总是到考前才顾得上来写个公众号福利。

如果要想提前听我押题，那就只有一个选择了，来找我学信奥。

真的不是要插广告。

一般广告都在最后面。。。

主要平时我事儿太多，不光只有信奥一件工作，还有好多开发的任务，所以老是想不起来写公众号。

所以如果在我的队里，那该上课的时候，我顺手就押题了。

断然不会到今天等我押题。

总之。

1- 初中联赛复赛考试特点

J组押题，有个特点，那就是，不需要抓某一个知识点，要抓某几个知识点的合集。

初中联赛的题目，因为不能涉及到太复杂的算法，所以只可能组合简单知识点成为一道题，所以单押一个知识点没用，要押，就押一堆。

不可能不考。

当然，动态规划这种看门虎不算，单个拿出来都是重中之重，得单出，大几率会捎上一个纯粹的DP。

所以知识点如下。

2- 押题知识点

如下：

1- 文件读写

这个是最基本的，如果不会文件读写，每次做题都是cin来cin去，考试也就开个freopen，还是继续用cin玩儿，那我觉得，考试不是不能过，但挺悬。

（所以您家娃是这样吗？如果有这个症状，不妨联系我们的咨询老师）

上面这个是广告。

所以说，cin不是不能用，而是要慎用。

初学的时候，用cin绝对可以。

但一次要读100个数，0或者1，也用cin嘛？

这么有时间？

一个一个手输？

再说了，cin里面的c，是console也就是终端。

所以cin的意思，是终端输入。

要是指向成为文件输入，那c是什么意思？current？

当前的输入？

也行吧，那为什么不把中文字倒过来写呢，反正最后你倒过来看就是了。

总之，不管是freopen，还是fstream，选一个用，用好了，就行。

话说这里提醒新人考生一句：

CSP-J可不是洛谷刷题，cin和cout那妥妥零分回家。

2- 基础数据类型

数据类型可不是简单的int还是double的区别，也就是整数和实数。

实际上，初中联赛或者高中联赛也很少考double。

因为double不准。

就算是有小数需要处理，也是用int来模拟一下。

这里说的数据类型，一般来说，是int和char的选择。

至于为什么要用int或者为什么要用char，就不展开了。

在这里，勉强算上个string，虽然说string并不是基础数据类型，但禁不住它简单好用。

3- 循环

这里有可能是for有可能是while，但绝不可能是老旧的do while。

只有初赛才会考do while这种破题。

总之循环。

到时候具体用哪个，按照你教练教的做就行了。

啥？教练没说具体什么时候用什么？

那完了。。。

总之，循环。

4- 普通线性数据结构

数据结构其实可以降维到理解string。string理解了，线性数据结构也就理解了。

具体怎么理解。

今天不讨论。

5- 进阶线性数据结构

初赛经常考的一道题，就是stack。

复赛可不经常考。

如果要考了，那就是涉及到算法优化。

所以勉强还是押上。

这里面还涉及哈希和队列。

就不展开了。 

6- 基础运算符

+-*/加个求余的%

没啥好说的，对求余敏感一点就行。

7- DP

来个基础的，那就是积水问题。

来个中级但是非常容易背诵的，那就是背包。

再高级一点的，不应该出现在初中联赛。

搜索什么的我就不押了，再押，整个考纲都说了。

3- 知识点组合

前面说了几个单拿出来，都是考的知识点（具体考哪个不一定）。

组合起来之后，第一题就可以押了，绝对中：

1 + 2 + 3 + 4

读文件 + 数据类型 + 循环 + 数据结构

简单的例子就是：

例题：给你几个字符，让你找到里面的数字，加一起。

这和头几年连续两年的真题第一题都非常贴近。

一个是给你最多五个数字，数数一共多少个。

一个是给你一定是8个数字而且一定是0或者1让你求有多少1。

简单吧？

然后，再来组合组合，第二题也可以押了

1 + 2 + 3 + 4 + 5

读文件 + 数据类型 + 循环 + 数据结构 + 进阶数据结构

例题：给你一个连在一起的字符串，最多500万位，数出里面的字符各自有多少，反向输出。

可以用map也可以用哈希。都行。

头几年也有过更复杂的题：地铁票。

说的是每张地铁票可以送一张公车票，问最后花费。

这显然是要用个队列。

但要能想到是队列，还是需要最少两年的修炼才可以，从开始接触初赛算起。

4- 考试注意事项

该注意的事情，你家娃的教练都应该已经叮嘱过了吧？

我就不多说了。

今天就捡重要的，说一个：

最近几年的CSP-J组的出题风格，在和CSP本体不断靠近。

意思是什么呢？

就是说，CSP-J的题，大几率会有一道技术类的模拟题。

CSP以前介绍过，这是给大学生准备的软件考试。

他们的题，通常都是从大学计算机科学系的学习内容中，挑选出来一些科技知识，然后在这个科普级别的知识点上，埋入一个模拟算法。

找到这个算法，就能解。

比如：

真题：服务器有个IP格式；客户端有个IP格式；筛选出正确的IP。

差不多这个意思，原题是这个长度的50倍。

这样的题，要真是看不懂，那就算了。

如果能坚持看完，那么，看完的时候，就是看懂的时刻。

这个时刻，就是能写程序的时刻。

其他的注意事项，大大小小得有个20多条。凡是我的学生们，我都反复叮嘱无数遍了。

实在不想再说一次。

要真想听，那就明年暑假找我们。

或者现在找我们，为了明年。

最后祝大家考试会的都能对。

要是今年没过初赛的同学，那家长不妨现在联系我们。

明年的赛季，要在当下准备。
